🔥 Top 5 Correcciones Prioritarias

1
SEOHead usa el dominio incorrecto agency-cms.com
Impacto crítico — Todas las URLs canónicas y de Open Graph generadas para artículos y páginas del blog apuntan a https://agency-cms.com/... en lugar de https://seogrowthers.com/.... Google indexa el dominio equivocado y las vistas previas en redes sociales son incorrectas.
Cómo corregirlo — src/components/shared/SEOHead.jsx, línea 19
// ❌ Antes
const domain = 'https://agency-cms.com';

// ✅ Después
const domain = 'https://seogrowthers.com';
2
Sitemap no declarado en robots.txt — Google no lo autodescubre
El robots.txt tiene la URL del sitemap comentada. Aunque el sitemap existe (/sitemap.xml, 43 URLs), Google no lo encontrará automáticamente, lo que ralentiza la indexación de tus 43 páginas.
Cómo corregirlo — robots.txt (descomenta la línea del Sitemap)
User-agent: *
Allow: /

Sitemap: https://seogrowthers.com/sitemap.xml
3
www.seogrowthers.com no redirige al dominio principal — riesgo de contenido duplicado
https://www.seogrowthers.com/ devuelve HTTP 200 con contenido propio (servidor Hostinger), mientras que https://seogrowthers.com/ está en Vercel. Google puede tratar ambos como sitios distintos con el mismo contenido, dividiendo la autoridad de tu dominio.
Cómo corregirlo — Configurar redirección 301 en Hostinger/Vercel
# En Vercel (vercel.json) — añade esta regla:
{
  "redirects": [
    {
      "source": "/:path*",
      "has": [{ "type": "host", "value": "www.seogrowthers.com" }],
      "destination": "https://seogrowthers.com/:path*",
      "permanent": true
    }
  ]
}

# También configurar en el panel DNS de Hostinger:
# Añadir redirección 301 de www → no-www
4
Página de Servicios sin meta description, OG tags ni schema
La página /services solo tiene <title>. Sin meta description Google extrae texto aleatorio del contenido. Sin OG tags, el enlace compartido en WhatsApp/LinkedIn no muestra imagen ni descripción. Sin schema Service, no puede aparecer en resultados enriquecidos.
Cómo corregirlo — src/pages/Services.jsx, dentro del bloque <Helmet> (línea 71–73)
<Helmet>
  <title>Servicios de SEO, Desarrollo Web y Automatización | SEO Growthers</title>
  <meta name="description" content="Descubre nuestros servicios premium: Desarrollo Web de alto rendimiento, Marketing Digital con ROI real y Automatización con IA. Crece con SEO Growthers." />
  <meta property="og:title" content="Servicios Premium | SEO Growthers" />
  <meta property="og:description" content="Desarrollo Web, SEO, Marketing Digital y Automatización con IA. La agencia que impulsa tu crecimiento digital." />
  <meta property="og:image" content="https://seogrowthers.com/images/services/og-services.jpg" />
  <meta property="og:url" content="https://seogrowthers.com/services" />
  <meta property="og:type" content="website" />
  <meta name="twitter:card" content="summary_large_image" />
  <link rel="canonical" href="https://seogrowthers.com/services" />
  <script type="application/ld+json">{JSON.stringify({
    "@context": "https://schema.org",
    "@type": "Service",
    "name": "Servicios Digitales de SEO Growthers",
    "provider": { "@type": "Organization", "name": "SEO Growthers" },
    "description": "Web Development, Marketing Digital y Automatización con IA.",
    "url": "https://seogrowthers.com/services",
    "areaServed": { "@type": "Country", "name": "Argentina" }
  })}</script>
</Helmet>
5
Meta description del sitio demasiado corta (104 chars) y sin imagen OG en Homepage
La meta description del index.html tiene 104 caracteres (el ideal es 150–160). Además, la homepage no define og:image, lo que resulta en vistas previas vacías al compartir en redes sociales.
Cómo corregirlo — src/pages/Home.jsx, bloque <Helmet> (línea 55–65)
<!-- Añadir og:image y ampliar description -->
<meta name="description" content="SEO Growthers es la agencia experta en Web Development, SEO y Analytics. Potencia tu crecimiento digital con estrategias basadas en datos reales y resultados medibles." />
<meta property="og:image" content="https://seogrowthers.com/images/og-home.jpg" />
<meta property="og:image:width" content="1200" />
<meta property="og:image:height" content="630" />
<meta name="twitter:card" content="summary_large_image" />
<meta name="twitter:image" content="https://seogrowthers.com/images/og-home.jpg" />

Guía Maestra: Nomenclatura y URLs para SEO

\

Esta guía detalla las prácticas recomendadas y los errores más comunes al momento de nombrar las páginas de un sitio web y estructurar las URLs de sus recursos, basándose en las directrices de la Búsqueda de Google.

\ \

Cómo Nombrar Páginas Web (Enlaces de Título)

\

El título de una página o \"enlace de título\" es vital para que los usuarios comprendan el contenido y los motivos por los que ese resultado es pertinente para su búsqueda [1, 2]. Para asegurar que la nomenclatura de tus páginas sea de alta calidad, sigue estas prácticas recomendadas:

\
    \
  • Usa siempre el elemento title: Comprueba que existe un título en el elemento <title> de todas y cada una de las páginas de tu sitio web [1, 2].
    • \
  • Sé descriptivo y conciso: Escribe títulos claros y directos que definan el contenido [1, 2].
    • \
  • Incluye tu marca de forma inteligente: Añade el nombre de tu sitio de forma concisa al principio o al final del elemento <title>, separado por un delimitador claro (como un guion, dos puntos o una pleca) para evitar que parezca repetitivo [3, 4].
    • \
  • Destaca visualmente el título principal: Asegúrate de que el título principal de la página sea diferente del resto del texto y el más importante, por ejemplo, utilizando una fuente más grande o colocándolo en el primer lugar visible del elemento <h1> [3, 4].
    • \
  • Respeta el idioma y sistema de escritura: Utiliza el mismo alfabeto e idioma en el <title> que el empleado en el contenido principal de la página para que Google no elija un texto alternativo [3, 4].
    • \
\ \

Errores Comunes al Nombrar Páginas Web

\

La Búsqueda de Google puede modificar automáticamente los títulos en los resultados si detecta problemas en la página o si el nombre elegido no representa bien el contenido [5, 6]. Para evitarlo, asegúrate de no cometer estos errores frecuentes:

\
    \
  • Dejar títulos medio vacíos: Evita estructuras donde falte el nombre del producto o contenido, como por ejemplo \"<title> | Nombre del sitio </title>\" [7, 8].
    • \
  • Utilizar títulos obsoletos: Si actualizas una página con información periódica (como un evento anual), no olvides actualizar la fecha en el elemento <title>. Dejar un año pasado es un error común [7-10].
    • \
  • Redactar títulos imprecisos: No uses títulos generales si el contenido de la página es muy específico, ya que Google intentará corregirlo si detecta que no refleja el contenido real [9-12].
    • \
  • Usar textos genéricos o vagos: Evita nombres no descriptivos como \"Página principal\" para tu inicio o \"Perfil\" para la sección de usuario [1, 2].
    • \
  • Incurrir en el exceso de palabras clave: Repetir la misma palabra o frase sin sentido (ej. \"Título, titulo, títulos\") no aporta valor y puede hacer que tu sitio sea percibido como contenido engañoso [1, 2].
    • \
  • Repetir textos o usar formato de plantilla: No uses el mismo título para todas tus páginas (ej. \"Productos a buen precio\") ni utilices textos excesivamente largos donde solo cambie un pequeño fragmento entre URL y URL [13, 14]. Es preferible actualizar dinámicamente el <title> [3, 4].
    • \
  • Falta de un título principal claro: Tener varios encabezados grandes y destacados con el mismo nivel o estilo confunde a los buscadores a la hora de identificar el nombre real de la página [15, 16].
    • \
  • Incluir precios muy variables: No agregues en el título precios que cambien con mucha rapidez (como los precios de vuelos), ya que pueden desactualizarse en los resultados y mostrar información errónea [3, 4].
    • \

Nomenclatura de Archivos y URLs de Imágenes

\

El nombre de los archivos y cómo estructuras sus URLs es esencial para que Google extraiga pistas sobre la temática de la imagen y las posicione de manera adecuada en la búsqueda visual [17].

\
    \
  • Nombres de archivo descriptivos: Utiliza siempre nombres que sean cortos y descriptivos. Un nombre como \"my-new-black-kitten.jpg\" es infinitamente superior a nombres autogenerados como \"IMG00023.JPG\" [17].
    • \
  • Cero nombres genéricos: Evita en la medida de lo posible nombrar tus URLs visuales con términos genéricos como \"image1.jpg\", \"pic.gif\" o \"1.jpg\" [17].
    • \
  • Traducción de URLs: Si vas a localizar tus imágenes para sitios internacionales, asegúrate de traducir también los nombres de los archivos, respetando siempre las directrices de codificación de URLs para caracteres especiales o no latinos [17].
    • \
  • Consistencia de URLs para el rastreo: Si utilizas la misma imagen en varias páginas de tu sitio, haz referencia a ella siempre mediante la misma URL. Esto permite que Google la almacene en caché de forma eficiente y no agote innecesariamente tu presupuesto de rastreo [18].
    • \
  • Indexación en URLs externas: Si tus imágenes se alojan en un dominio distinto mediante una Red de Distribución de Contenido (CDN), asegúrate de utilizar un sitemap de imágenes para incluir estas URLs externas y garantizar que Google las asocie correctamente con tu página web [19-21].
    • \
